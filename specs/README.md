# Spec Corpus

Execution plans and companion specs live here, separate from:

- [`../ontology/README.md`](../ontology/README.md): source of truth for project ontology
- [`../doc/`](../doc/): maintainer notes, runbooks, and product-facing contracts

Plan files in this directory are execution/history artifacts. If an older plan
repeats noun definitions or topic-centric chains that conflict with
[`../ontology/`](../ontology/), the ontology folder wins. New plan work should
use the current control-surface vocabulary unless it is explicitly quoting
historical text.
